# 1. 文書概要

本書は、組み込みAIソリューション検証プロジェクトにおけるリスクを整理し、対応方針を定義することを目的とする。

本プロジェクトでは、エッジAIデバイス、AI SDK、周辺機器、クラウドサービスなど複数のOSSおよびハードウェアを組み合わせて検証を行うため、技術的なリスクを継続的に管理する。

# 2. リスク一覧

| ID    | リスク                                   | 影響 | 発生確率 | 対応方針                                  | 状態  |
| ----- | ------------------------------------- | -- | ---- | ------------------------------------- | --- |
| R-001 | AI SDKのバージョンアップにより手順やAPIが変更される        | 高  | 中    | 利用バージョンを設計書へ記載し、変更点をブログへ反映する          | 管理中 |
| R-002 | AIモデルのライセンス・提供条件が変更される                | 中  | 低    | ライセンスを確認し、代替モデルを調査する                  | 管理中 |
| R-003 | 評価ボード（Kakip）の販売終了・供給終了                | 高  | 低    | RZ/Vシリーズ評価ボードや後継機種への移行を検討する           | 管理中 |
| R-004 | USBカメラがLinuxまたはAI SDKで正常に利用できない       | 高  | 中    | USB認識、画像取得、AI推論を段階的に検証し、動作確認済み機器を管理する | 管理中 |

| R-011 | AI SDKやサンプルアプリが想定するハードウェア構成と検証環境が異なる  | 中  | 高    | 構成差分を設計書・ブログへ記録し、再現手順として公開する          | 管理中 |

# 3. リスク対応方針

本プロジェクトでは、以下の方針でリスク管理を行う。

* 利用するSDKやOSSのバージョンを設計書へ記録する。
* 検証結果や回避策はブログおよびGitHubへ公開し、再現可能な技術情報として蓄積する。
* ハードウェアや周辺機器は段階的に検証し、動作確認済み構成を管理する。
* 特定製品への依存を避け、代替製品への移行を考慮した設計とする。
* クラウドサービスや通信方式は抽象化し、将来的な変更に対応できる構成とする。

# 4. リスク監視

リスク管理は、各マイルストーン（MS-001～MS-011）の完了時に見直しを行う。

新たなリスクが判明した場合は、本書へ追加し、影響度・発生確率・対応方針を更新する。

# 5. まとめ

本プロジェクトでは、技術検証で得られた知見を設計書・ブログ・GitHubへ反映し、リスクそのものを技術資産として蓄積する。

継続的にリスク管理を実施し、再現性の高い検証環境を維持するとともに、将来的な設備向けAIソリューションへの展開を目指す。
