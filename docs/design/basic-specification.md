# 組み込みAIソリューション検証プロジェクト 基本仕様書

# 1. 文書概要

本書は、組み込みAIソリューション検証プロジェクトの基本仕様を整理する。

本プロジェクトでは、エッジAIデバイスを利用した画像認識システムを対象とし、AI推論結果を設備機器、Webシステム、クラウドサービスへ連携するための基本構成を定義する。

本書では製品選定理由および基本構成を整理し、詳細な実装方法については詳細設計で検討する。

# 2. システム構成

初期構成は以下を想定する。

```text
USB Camera
      │
      ▼
Kakip
(Renesas RZ/V)
      │
      ▼
AI SDK
      │
      ▼
AI推論
      │
      ├──── PLC通信
      ├──── Web表示
      └──── クラウドサービス
```

# 3. AIプラットフォーム

## 3.1 採用製品

Renesas RZ/Vシリーズ

### 採用理由

* AI SDKが提供されている
* 学習済みモデルが提供されている
* サンプルアプリが充実している
* 転移学習による追加学習を想定できる
* エッジAI向けSoCとして実績がある

AIモデルを一から開発することを目的とせず、既存資産を利用して短期間でPoCを実施することを優先する。

# 4. AIデバイス

## 4.1 採用製品

Kakip

### 採用理由

Renesas製SoCを利用した独自ハードウェアを開発するには、

* 基板設計
* 電源設計
* DDR設計
* Linux BSP構築

など、多くの開発工数を要する。

そのため、本プロジェクトではRenesas RZ/Vシリーズを採用したKakipを仮採用する。

### 将来方針

Kakipが販売終了または供給終了となった場合は、

* RZ/Vシリーズ評価ボード
* 独自基板
* 他社エッジAIデバイス

への移行を検討する。

# 5. AIモデル

初期検証ではSDK付属モデルを利用する。

独自AIモデルについては、

* 転移学習
* 少量データ学習

を基本方針とする。

完全自社開発モデルについては本プロジェクトの対象外とする。

# 6. 開発環境

## ホストPC

* Windows 11

## Linux環境

* WSL2

## コンテナ

Docker環境を利用する。

AI SDKについてはDockerコンテナ上でビルド・開発を行い、Kakip本体はAI推論実行環境として利用する。

ホスト環境を汚さず、再現性のある開発環境を維持することを目的とする。

# 7. PLC連携

AI推論結果をFA設備へ連携するため、PLCとの通信を検討する。

初期検証では通信検証環境を利用し、

* 通信方式
* データ形式
* 更新周期

を確認する。

通信プロトコルについては設備に応じて選択可能とする。

# 8. Web表示

KakipにはUbuntu Linuxが搭載されている。

そのため、AI推論結果の表示については.NET（Blazor）を利用したWebアプリケーションを基本構成として検討する。

表示対象例

* 推論結果
* 検出個数
* 判定結果
* 最新画像
* 推論履歴

# 9. クラウドサービス

AI推論結果をクラウドサービスへ送信し、

* データ保存
* 遠隔監視
* ダッシュボード表示

を実現できる構成を検討する。

初期検証ではCloudflare Workersを採用候補とする。

### 採用理由

* 無料利用枠が広い
* API構築が容易
* Queue等の非同期処理が利用できる
* D1等との親和性が高い
* 小規模PoCに適している

将来的には用途に応じて他クラウドサービスへの移行も可能な構成を目指す。

# 10. AI推論結果

AI推論結果として以下を取得対象とする。

* 検出対象
* 個数
* 信頼度
* OK/NG判定
* 推論時刻

取得した情報は設備連携およびWeb表示へ利用する。

# 11. 非機能要件

* Dockerによる再現可能な開発環境
* OSSを積極的に活用する
* 特定製品への依存を最小限とする
* クラウドサービスを利用する場合もローカル動作を考慮する
* AIモデルを容易に差し替え可能とする

# 12. 将来構想

今後、以下の拡張を検討する。

* 転移学習による独自AIモデル作成
* PLC実機との通信
* タッチパネル連携
* シグナルライト制御
* クラウドサービスとの双方向通信
* Webダッシュボード
* AI設備監視システム
* AI外観検査システム
* AI個数カウントシステム

# 13. まとめ

本プロジェクトでは、既存AI SDKを活用しながら、AI推論結果を設備機器、Webシステム、クラウドサービスへ展開可能なソフトウェア構成を検証する。

各製品は現時点での採用候補とし、将来的に代替可能な構成を前提とする。

本基本仕様書を基に、詳細設計および各技術要素の検証を進める。
